# ğŸ”¥ **What this diagram shows (in simple words)**

Your system does NOT simply "reuse" refresh tokens.

It does this instead:

### 1. **Every refresh token is one-time-use**  
Once itâ€™s used, it is revoked.

### 2. **Server stores only a hashed version**  
Stolen tokens are useless without knowing the exact string.

### 3. **Each refresh creates a NEW refresh token**  
This prevents replay attacks.

### 4. **Rotation chain detects theft**  
If an attacker tries to use an older token:

- You detect it  
- You revoke all tokens in chain  
- You stop the attacker immediately  
- User must re-login  

### 5. **IP tracking adds extra detection**  
You can see:

- Same user  
- Two refresh requests  
- Two different IPs  
- Seconds apart  

This is a theft alarm.
